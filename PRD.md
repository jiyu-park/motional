# MOTIONAL PRD

| 항목 | 내용 |
| --- | --- |
| 제품 | MOTIONAL |
| 버전 | MVP |
| 플랫폼 | React Native(Expo), iOS·Android |
| 기준 자료 | `ref/` HTML 프로토타입 |

## 1. 제품 소개

MOTIONAL은 사용자가 현재 감정을 간단히 기록하고, 캘린더와 통계로 감정 흐름을 확인하며, 감정에 맞는 음악을 추천받는 모바일 앱이다.

긴 일기 대신 **감정 선택 → 활동·메모 입력 → 저장**의 짧은 흐름을 제공한다. 의료 진단이나 상담이 아닌 일상적인 자기 관찰 도구를 목표로 한다.

## 2. MVP 목표

- 사용자가 1분 안에 감정 기록을 완료할 수 있다.
- 저장한 기록이 앱을 종료한 뒤에도 유지된다.
- 날짜별 기록과 기본 통계를 쉽게 확인할 수 있다.
- 최근 감정에 맞는 mock 플레이리스트를 확인할 수 있다.

## 3. 핵심 사용자 흐름

```text
캘린더 또는 홈
→ 감정 기록 시작
→ 감정 선택
→ 활동 태그와 메모 입력(선택)
→ 저장
→ 캘린더 반영
→ 감정별 추천 음악 확인
```

## 4. 핵심 기능

### FR-01 감정 기록

- 사용자는 감정 1개를 선택한다.
- 활동 태그는 여러 개 선택할 수 있다.
- 메모는 선택 입력이며 최대 500자다.
- 감정을 선택하지 않으면 저장할 수 없다.
- 저장 완료 후 캘린더에서 기록을 확인할 수 있다.

### FR-02 기록 저장 및 관리

- Zustand로 감정 기록 상태를 관리한다.
- AsyncStorage에 기록을 저장한다.
- 앱을 다시 실행해도 기록이 유지되어야 한다.
- 사용자는 기존 기록을 조회, 수정, 삭제할 수 있다.
- 저장 데이터가 손상되어도 앱이 종료되지 않고 빈 상태로 복구한다.

### FR-03 감정 캘린더

- 월간 캘린더에 기록이 있는 날짜의 감정을 표시한다.
- 이전 달, 다음 달, 오늘로 이동할 수 있다.
- 날짜를 선택하면 해당 날짜의 기록을 보여준다.
- 기록이 없는 날짜에서는 새 감정 기록을 시작할 수 있다.

### FR-04 음악 추천

- 최근에 저장한 감정을 기준으로 mock 플레이리스트를 표시한다.
- 추천 항목에는 곡명, 아티스트, 이미지 또는 아이콘을 표시한다.
- 활동 태그를 선택하면 관련 mock 목록을 보여준다.
- MVP에서는 실제 음악 API와 전체 음원 재생을 구현하지 않는다.

### FR-05 기본 통계

- 최근 7일과 30일의 감정 기록을 조회할 수 있다.
- 가장 자주 기록한 감정과 감정별 횟수를 표시한다.
- 날짜별 감정 흐름과 자주 선택한 활동을 표시한다.
- 기록이 부족하면 그래프 대신 안내 문구를 보여준다.

### FR-06 홈

- 오늘의 감정 기록 상태와 기록 버튼을 가장 먼저 표시한다.
- 최근 감정에 맞는 대표 mock 플레이리스트 1개를 표시한다.
- 이번 주 요약과 최근 기록을 간단히 보여준다.
- 상세 정보는 캘린더, 음악, 통계 화면으로 이동해 확인한다.
- 우측 상단 Profile 아이콘으로 설정 화면에 이동한다.

## 5. 화면 구성

| 화면 | 주요 역할 | 참고 파일 |
| --- | --- | --- |
| 홈 | 오늘의 체크인, 음악 추천, 주간 요약, 최근 기록 | 신규 구현 |
| 캘린더 | 월간 기록 확인, 날짜 선택, 기록 시작 | `ref/mood_calendar.html` |
| 감정 기록 | 감정·활동·메모 입력과 저장 | `ref/mood.html` |
| 음악 | 최근 감정과 활동 기반 mock 추천 | `ref/mood_playlist.html` |
| 통계 | 감정 흐름, 분포, 활동 요약 | `ref/dashboard.html` |
| 설정 | 앱 정보와 로컬 데이터 관리 | 우측 상단 Profile 아이콘 |

하단 탭은 `Home`, `Calendar`, `Music`, `Stats` 순서로 통일한다. Profile은 각 탭의 우측 상단 아이콘에서 설정 화면으로 이동한다.

## 6. 데이터 모델

```ts
export type MoodEntry = {
  id: string;
  date: string; // YYYY-MM-DD
  mood: string;
  activities: string[];
  note?: string;
  createdAt: string; // ISO 문자열
  updatedAt: string; // ISO 문자열
};
```

동일 날짜에 여러 기록을 저장할 수 있다. 캘린더에는 해당 날짜의 마지막 기록을 대표 감정으로 표시한다.

## 7. UX 원칙

- `ref/`의 파스텔 색상, 둥근 카드, 감정 캐릭터 분위기를 유지한다.
- 기본 UI 언어는 한국어로 통일한다.
- 감정을 좋고 나쁨으로 평가하지 않고 중립적으로 표현한다.
- 작은 화면, Safe Area, 키보드 노출 상황에서도 주요 버튼을 사용할 수 있어야 한다.
- 색상만으로 감정을 구분하지 않고 텍스트나 아이콘을 함께 사용한다.
- 아이콘 버튼에는 접근성 레이블을 제공한다.

## 8. MVP 완료 기준

- 감정 기록 생성, 조회, 수정, 삭제가 동작한다.
- 앱 재실행 후에도 AsyncStorage 기록이 유지된다.
- 캘린더, 음악, 통계 화면에 저장 데이터가 반영된다.
- 화면 이동과 하단 탭이 정상 동작한다.
- 기록 없음, 저장 실패, 손상 데이터 상태를 처리한다.
- `npm run lint`, `npx tsc --noEmit`, `npx expo-doctor` 검사를 통과한다.

## 9. 제외 범위

- 회원가입, 로그인, 서버 동기화
- 실제 AI 추천 API
- 실제 음악 스트리밍 및 유료 음악 서비스 연동
- 사용자 간 공유, 소셜 기능
- 의료 진단, 상담, 치료 기능
- 푸시 알림과 PDF 내보내기
